# LLM Phish Sphere Prediction Contest

## What this project is
A competition between 6 AI language models predicting the setlists for Phish's 
9-show Las Vegas Sphere residency (April–May 2026). Models are scored on accuracy, 
confidence calibration, and optional set placement predictions. Results are 
displayed on a public website.

## Show schedule
- Weekend 1: Thu Apr 16, Fri Apr 17, Sat Apr 18 (shows 1–3)
- Weekend 2: Thu Apr 24, Fri Apr 25, Sat Apr 26 (shows 4–6)
- Weekend 3: Thu Apr 30, Fri May 1, Sat May 2 (shows 7–9)

## Competing models
- Claude (Anthropic, claude-sonnet-4-6)
- ChatGPT (OpenAI, GPT-4o)
- Gemini (Google, Gemini 1.5 Flash)
- Perplexity AI
- Grok (xAI)
- Mistral AI

## How predictions are collected
Each model receives an identical prompt and returns predictions in CSV format.
Predictions are copy-pasted (or automated where possible) into the data folder.
One CSV file per model per weekend. Actual setlists are entered after each show.

## Scoring rules
- Base score: 10 points per correct song in the correct show
- Set bonus: +2 pts for correct set placement (Set1 or Set2)
- Position bonuses: +3 pts for correct set opener, +3 pts for correct set closer
- Encore bonus: +5 pts for correct encore song
- Confidence multiplier: score × (Confidence / 3)
  - Confidence 5 = 1.67× multiplier
  - Confidence 3 = 1.0× multiplier  
  - Confidence 1 = 0.33× multiplier
- Revision penalty: changing a confidence 4–5 pick in a later weekend = 50% loss 
  of confidence bonus for that pick

## Ranking dimensions
1. Primary: total points across all 9 shows
2. Secondary: points per weekend (3-show block)
3. Tertiary: raw correct song percentage (unweighted)

  /scores
    scores.json        (auto-generated by scoring script)

## CSV formats

### Prediction file columns
Weekend_Number, Show_Number, Show_Date, Song_Title, Confidence, Set_Placement
(Set_Placement is optional: Set1, Set2, Encore, or blank)

### Actual setlist file columns
Show_Number, Show_Date, Song_Title, Set_Placement, Position
(Position: Opener, Closer, Encore, or blank for mid-set)

## Website requirements
The website reads from /data and displays:

1. **Overall leaderboard** — total points per model, all 9 shows
2. **Weekend leaderboard** — points per model per 3-show weekend
3. **Per-show detail** — for each show: predicted songs vs. actual, 
   which models got it right, points earned
4. **Weekend summary view** — the 3-show block as a unit, 
   weekend winner highlighted
5. **About / How It Works page** — full scoring explanation for visitors

## Automation goals
- Scoring script reads prediction CSVs + actual CSVs and writes scores.json
- Website reads scores.json (no manual score entry)
- Prediction ingestion: manual CSV paste for now, automate later if possible

## Owner workflow per weekend
1. Send prompt to all 6 models
2. Copy CSV output into /data/predictions/weekendN_modelname.csv
3. After each show: enter actual setlist into /data/actuals/showN_actual.csv
4. Run scoring script (or have it run automatically)
5. Website updates from new scores.json

## Tech stack
- Frontend: Next.js (React)
- Hosting: Vercel (free tier)
- Data: flat CSV files + generated JSON (no database)
- Scoring: Node.js script in /scripts/score.js
- Version control: GitHub (required for Vercel deployment)
